## Feature Split

### Free Tier
- **Themes**: Cyberpunk + Midnight (2 of 5)
- **Zoom**: 10–14
- **Heatmap**: Yes (full quality)
- **Export**: SDR only, full resolution, save + share
- **Location modes**: All (auto, address, photos)

### Pro Tier (unlocked by IAP or legacy purchase)
- **Themes**: All 5 (adds Ember, Ghost, Moss)
- **Zoom**: 15–16 (overzoom street detail)
- **HDR**: 10-bit HEIF export
- **Future features**: All new themes/features go to Pro

## Implementation Steps

### 1. Theme Gating (`Sources/MapCore/Theme.swift`)

Add free/pro classification to themes:

```swift
public struct MapTheme {
    // ... existing fields ...
    public let isPro: Bool  // new
}
```

Update theme initializers:
- cyberpunk, midnight: `isPro: false`
- ember, ghost, moss: `isPro: true`

### 2. PaywallView (`iOS/PaywallView.swift`) — NEW FILE

Simple sheet showing:
- App icon + "Cartogram Pro"
- Feature list (all themes, HDR, zoom 15-16)
- Price from `StoreManager.proProduct?.displayPrice`
- "Unlock Pro" button → `StoreManager.purchasePro()`
- "Restore Purchases" link → `StoreManager.restorePurchases()`
- Dismiss (X) button

Design: dark background, match app aesthetic. No subscription — single purchase, clear and simple.

### 3. Settings Gating (`iOS/GeneratorSettingsView.swift`)

**Theme section:**
- Pro themes show a lock icon overlay on the swatch
- Tapping a locked theme opens PaywallView instead of selecting it
- After purchase, theme is selected automatically

```swift
ThemeOptionRow(
    theme: theme,
    isSelected: theme.id == viewModel.selectedThemeId,
    isLocked: theme.isPro && !store.isPro
) {
    if theme.isPro && !store.isPro {
        showPaywall = true
    } else {
        viewModel.selectedThemeId = theme.id
    }
}
```

**HDR toggle:**
- Show toggle but disabled when not Pro
- Label shows "Pro" badge
- Tapping when locked opens PaywallView

### 4. Zoom Gating (`iOS/ContentView.swift`)

- Free: `canZoomIn: viewModel.zoom < 14` (was 16)
- Pro: `canZoomIn: viewModel.zoom < 16`
- Same for pinch gesture: clamp to 14 unless Pro
- When user hits the free limit, brief toast "Unlock Pro for street-level zoom"

### 5. Export Gating (`iOS/GeneratorViewModel.swift`)

- Free users: Force `hdrEnabled = false` in `generate()` even if the toggle was on
- The HDR code path is only entered when `isPro && hdrEnabled`
- No resolution degradation — keep the free experience high quality

### 6. Reset Pro Theme on Downgrade

If a user selects a Pro theme then somehow loses Pro status (refund):
- In `refreshProStatus()`, if `!isPro` and `selectedThemeId` is a pro theme, reset to "cyberpunk"

### 7. StoreKit Testing

- Add `iOS/CartogramPro.storekit` StoreKit configuration file
- Test scenarios:
  - Fresh install (free) → purchase Pro → themes/HDR/zoom unlock
  - Legacy paid user → auto-Pro on launch
  - Restore purchases on new device
  - AppTransaction fallback (user who paid but missed v1.1)
